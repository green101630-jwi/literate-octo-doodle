Featured Project

Dual-arm OMX Sim-to-Real Robot Learning Pipeline

In Progress

This project builds a sim-real robot learning pipeline for a dual-arm OpenManipulator-X system. The pipeline connects real teleoperation with IsaacSim/IsaacLab simulation, records /pair1 and /pair2 joint trajectories as reusable episodes, organizes teleoperation and scripted demonstrations into learning-ready datasets, and applies ACT-based behavior cloning with phase/image ablation experiments. Hardware mapping experiments are currently in progress.

IsaacSim/IsaacLab replay, grasp behavior comparison, and physics/contact setting analysis.

Behavior cloning setup using qpos, image observations, phase information, action chunks, and normalization statistics.

Module 1

Sim-real Teleoperation System

Built

Built a sim-real teleoperation pipeline that links real dual-arm OMX teleoperation motion with simulated robots in IsaacSim and IsaacLab. Joint states from ROS2 /pair1 and /pair2 are recorded by episode and replayed for simulation validation.

Implemented Components

  • ROS2-based recording/replay structure for dual-arm joint trajectories.
  • Episode-level storage of /pair1 and /pair2 joint states.
  • Replay pipeline for connecting real teleoperation motion to simulation robots.
  • Approximately 1,377 joint trajectory samples confirmed from real teleoperation episodes.

Simulation Analysis

  • Cube grasping was feasible in IsaacSim.
  • Grasp behavior was unstable in IsaacLab, requiring physics/contact setting analysis.
  • Real OMX gripper uses a 1-joint command.
  • Simulation gripper uses a 2-joint representation, requiring gripper mapping analysis.

Module 2

Teleop and Scripted Demonstration Collection

Built

Built a demonstration collection pipeline using both real robot teleoperation and scripted trajectory generation. Teleoperation demonstrations were organized into HDF5/NPZ and robomimic-style dataset formats, and approximately 94 demonstrations were reviewed as behavior cloning candidates.

Teleop Demonstration Task

  • Task: two cubes are placed into cups, with each arm handling one cube.
  • Data source: real dual-arm OMX teleoperation.
  • Dataset format: HDF5/NPZ and robomimic-style demonstration structure.
  • Approximately 94 teleoperation demonstrations reviewed as BC training candidates.

Scripted Demonstration Task

  • Task: place the yellow cube on top of the green cube.
  • Task phases: approach, grasp, lift, place, and retreat.
  • Script generation: phase-wise waypoints, inverse kinematics, and minimum-jerk trajectory.
  • Validation points: end-effector definition, gripper link position, grasp pose, and object randomization.

Module 3

ACT-based Behavior Cloning and Ablation Study

Experimenting

Analyzed the ACT/ALOHA behavior cloning structure, where qpos and image observations are used to predict future action chunks through a Transformer-based policy. The collected OMX demonstrations were adapted into an ACT-style training pipeline by organizing qpos, action targets, episode length, action chunks, normalization statistics, and rollout conditions.

01

ACT Structure

Transformer-based behavior cloning for future action chunk prediction.

02

Dataset Statistics

dataset_stats.pkl based qpos/action normalization and inference-time denormalization.

03

Phase Ablation

Comparison between policies with and without task phase conditioning.

04

Image Ablation

Comparison between single-camera image input and joint/qpos-only input.

Ablation Conditions

  • With phase input versus without phase input.
  • With one camera image versus without image observation.
  • Joint/qpos-only policy comparison.
  • Policy rollout comparison across input conditions.

Learning Pipeline Checks

  • qpos/action target formatting.
  • Episode length and action chunk consistency.
  • Normalization and denormalization flow.
  • Simulation rollout evaluation setup.

Module 4

Trained Policy Hardware Mapping

Experiments In Progress

Hardware mapping experiments are currently in progress. The current focus is to align simulation observations with real robot observations and to check whether trained policy outputs can be transferred to the real OMX setup in a stable and consistent manner.

Mapping Items Under Test

  • qpos, phase, and image input consistency between simulation and real hardware.
  • Joint order, action scaling, joint limits, and command frequency checks.
  • 2-joint simulation gripper representation to 1-joint real gripper command conversion.

Stepwise Validation

  • Pair1/pair2 follower controller execution checks.
  • FollowJointTrajectory action server verification.
  • First-point test before long-horizon rollout.
  • Short-horizon rollout tests before full real-world evaluation.

Technical Challenges

Key Issues Analyzed

IsaacSim vs IsaacLab

Grasping is possible in IsaacSim, while unstable grasp behavior appears in IsaacLab.

Gripper Mismatch

Real 1-joint gripper command differs from the simulated 2-joint gripper representation.

Observation Consistency

qpos, phase, image input, scaling, and frequency must remain consistent across simulation and real hardware.

Hardware Safety

First-point and short-horizon tests are required before long-horizon real robot rollout.
